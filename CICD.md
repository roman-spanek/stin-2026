# how to set up CI/CD for a project using GitHub Actions

1. Create a GitHub repository for your project if you haven't already.
2. In your repository, create a new directory called `.github/workflows`. This is where you will store your GitHub Actions workflow files.
3. Inside the `.github/workflows` directory, create a new file called `ci.yml` (you can name it anything you like, but `ci.yml` is a common convention).
4. Open the `ci.yml` file and add the following content to define your CI/CD as in .github/workflows/ci.yml

```yaml
name: CI Pipeline

on:
  push:
    branches:
      - "feature/**"
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Setup Java
        uses: actions/setup-java@v3
        with:
          distribution: 'temurin'
          java-version: '21'

      - name: Build with Maven
        run: mvn clean install

      - name: Run tests
        run: mvn test

```
5. Commit and push the `ci.yml` file to your repository.
6. GitHub Actions will automatically trigger the workflow whenever you push changes to branches that match the specified patterns (e.g., `feature/**` or when a pull request is opened against `main`).
7. You can monitor the progress of your CI/CD pipeline in the "Actions" tab of your GitHub repository. Here you can see the logs for each step of the workflow and identify any issues that may arise during the build and test process.

#add pipeline with test coverage added to pull request
1. To add test coverage reporting to your CI/CD pipeline, you can use a tool like JaCoCo for Java projects. First, you need to configure JaCoCo in your Maven `pom.xml` file:

```xml
<plugin>
				<groupId>org.jacoco</groupId>
				<artifactId>jacoco-maven-plugin</artifactId>
				<executions>
					<execution>
						<id>prepare-agent</id>
						<goals>
							<goal>prepare-agent</goal>
						</goals>
					</execution>
					<execution>
						<id>report</id>
						<phase>verify</phase>
						<goals>
							<goal>report</goal>
						</goals>
					</execution>
				</executions>
			</plugin>
```
2. Next, you can modify your `ci-test-coverage.yml` workflow file to include a step for generating the test coverage report and uploading it as an artifact:

```yaml
name: CI Pipeline with test coverage

on:
  push:
    branches:
      - "feature/**"
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Setup Java
        uses: actions/setup-java@v3
        with:
          distribution: 'temurin'
          java-version: '21'

      - name: Build and test with coverage
        run: mvn clean verify # This will run the tests and generate the JaCoCo report, do not run with install as it will skip tests

      - name: Upload JaCoCo report
        uses: actions/upload-artifact@v4
        with:
          name: jacoco-report
          path: target/site/jacoco
          
      - name: Add coverage to PR
        uses: madrapps/jacoco-report@v1.6.1
        with:
          paths: target/site/jacoco/jacoco.xml
          token: ${{ secrets.GITHUB_TOKEN }}
```
3. Commit and push the updated `ci-test-coverage.yml` file to your repository.
4. Now, when you push changes to branches that match the specified patterns or open a pull request against `main`, the workflow will run, and you will see the test coverage report generated by JaCoCo. The report will be uploaded as an artifact, and you can also see the coverage information directly in the pull request using the `madrapps/jacoco-report` action. This will help you ensure that your code changes maintain or improve test coverage, and it provides visibility into the quality of your codebase.
5. You can further customize the workflow to include additional steps, such as deploying your application or sending notifications based on the results of the CI/CD pipeline.
